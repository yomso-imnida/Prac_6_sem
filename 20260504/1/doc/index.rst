MOOD documentation
==================

MOOD - многопользовательская игра, где игроки перемещаются
по закольцованному полю, добавляют монстров, атакуют их и обмениваются
сообщениями.

Установка
----------

Клонирование репозитория::

   git clone <repository_url>
   cd <repository_directory>

Создание виртуального окружения::

   python3 -m venv .venv
   source .venv/bin/activate

Установка зависимостей для разработки::

   pip install -r requirements-dev.txt

Сборка wheel-пакета::

   python3 -m doit wheel

Установка собранного wheel::

   pip install dist/mood_mud-0.1.0-py3-none-any.whl


Обычный запуск (1 вариант)
--------------------------

После установки wheel-пакета становятся доступны команды::

Запуск сервера::

   mood-server

Запуск клиента::

   mood-client <user_name>

Например

В одном терминале::

   mood-server

и в другом терминале::

   mood-client alice


Обычный запуск (2 вариант)
--------------------------

Запуск сервера::

   python3 -m mood.server

Запуск клиента::

   python3 -m mood.client <user_name>

Например запуск клиента::

   python3 -m mood.client alice

Имя пользователя передаётся серверу при подключении. Если такое имя уже
занято, то сервер отклоняет подключение.

Запуск (с файлом)
-----------------

Клиент может читать команды из файла::

   python3 -m mood.client <user_name> --file <commands_file>

Например::

   python3 -m mood.client alice --file test.mood

В этом режиме команды берутся из файла, обычный ввод через ``командную
строку`` не используется. Между отправками команд на сервер выдерживается
пауза (не менее 1 секунды).

Пример файла ``test.mood``::

   sayall "Hello from file"
   addmon dragon hello "Hi, I'm dragon :)" hp 30 coords 1 0
   right
   attack dragon with sword
   attack dragon with axe


Игровое поле
------------

Поле размером 10x10. Координаты закольцованы: если игрок выходит
за границу поля, то он появляется с противоположной стороны.

Например, переход вверх из клетки ``(0, 0)`` приводит в клетку ``(0, 9)``.


Команды для игрока
------------------

Движение игрока::

   up
   down
   left
   right

Добавление монстра::

   addmon <monster_name> hello <message> hp <hp> coords <x> <y>

Например::

   addmon dragon hello "Hi! I'm dragon" hp 30 coords 1 0

Атака монстра::

   attack
   attack <monster_name>
   attack with <weapon>
   attack <monster_name> with <weapon>

Доступное оружие::

   sword
   spear
   axe

Отправка сообщения всем игрокам::

   sayall <message>

Например::

   sayall Hi
   sayall "Let's attack dragon"

Смена локали сервера::

   locale <locale_name>

Например::

   locale ru_RU.UTF-8

Открытие документации::

   documentation

Управление перемещением монстров (статичные или бродячие)::

   movemonsters on
   movemonsters off


Монстры
-------

Монстр хранит имя, приветственную фразу и количество здоровья.

Если игрок попадает на клетку с монстром, сервер отправляет ему сообщение
с изображением монстра и его приветственной фразой.

Один раз в 30 секунд сервер перемещает случайного монстра на одну клетку
в случайном направлении.
Если новая клетка занята другим монстром, то выбирается другой ход.

При успешном перемещении монстра все игроки получают сообщение вида::

   dragon moved one cell right


Локализация
-----------

Сервер поддерживает локализацию сообщений с помощью gettext и Babel.

Игрок может изменить язык серверных сообщений командой::

   locale ru_RU.UTF-8

После смены локали часть сообщений сервера отображается на русском языке.


MOOD-чат
--------

Игроки могут обмениваться сообщениями через общий чат.

Сообщение отправляется командой::

   sayall "message"

Все подключённые игроки получают сообщение мгновенно.


Архитектура проекта
-------------------

Проект разделён на клиентскую и серверную части.

Серверная часть:
   - хранит состояние игрового поля
   - обрабатывает подключения клиентов
   - синхронизирует игроков
   - управляет монстрами

Клиентская часть:
   - предоставляет интерфейс командной строки
   - отправляет команды серверу
   - показывает ответы сервера

Для одновременной работы нескольких игроков используется asyncio.
